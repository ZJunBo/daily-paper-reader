Abstract 

As LLMs continue to scale, improving training efficiency increasingly depends on using data more effectively. Data selection addresses this problem by allocating a limited training budget to samples that best promote a target behavior. Existing methods usually represent the target behavior with a set of target examples, but often treat these examples as equally important. This can be inefficient because target examples may differ in their relevance to the current model: examples closer to the model’s current behavior provide more actionable guidance than those farther away. We propose PRISM (Pr eference-aware Influence-function-based Data Selection 

Method), which uses the current model’s prefer-ence to weight target examples and construct a preference-aware target representation. PRISM then scores candidate training samples by their alignment with this representation, concentrat-ing the data budget on samples more likely to move the model toward the target behavior. Theoretical analysis shows that this preference weighting yields a more effective first-order direction for increasing target-behavior prefer-ence. Experiments across model families and scales show that PRISM improves both effi-cient fine-tuning and safety-oriented SFT re-pair, demonstrating that precise target-behavior characterization is key to budget-efficient data selection.

function-based Data Selection Method). PRISM measures the preference for each target example by the likelihood that the current model generates the corresponding response, and uses this preference to weight each example when constructing the target direction. A higher preference indicates that an example corresponds to a nearby behavioral direc-tion that can be more readily strengthened through training. Training samples associated with such examples are therefore more likely to move the model toward the target behavior efficiently under a limited data budget. Conversely, lower-preference examples receive smaller weights because they lie farther from the model’s current behavioral distri-bution and provide weaker selection signals. PRISM then scores candidate samples by com-paring each training sample’s influence signal with the preference-aware target direction. Through this procedure, PRISM concentrates the data budget on samples that are most likely to move the model toward the target behavior, thereby improving the target capability more efficiently. Theoretically, we show that PRISM captures the first-order direction that most effectively increases target-behavior preference in the neighborhood of the current model. Under a fixed selection budget, selecting samples along this direction maximizes the desired gain in target preference per unit of data. By contrast, unweighted averaging mixes high- and low-preference target examples, causing the estimated direction to deviate from the behav-ioral pattern toward which the current model can most readily be pushed, thereby reducing the utility of each selected sample. We conduct systematic experiments across model families and scales in two complementary data selection settings. First, from the perspec-tive of improving general model capabilities, we instantiate the target behavior as knowledge abil-ity, mathematical reasoning, and safety guardrail ability. The results show that, under the same data budget, training samples selected by PRISM yield stronger fine-tuning performance. Second, in the safety-critical SFT repair setting, PRISM more ef-fectively mitigates harmful model behaviors under the same removal budget. Overall, these results demonstrate that PRISM improves data selection efficiency by providing a more precise characteri-zation of the target behavior. 

2 Related Work 

Training data attribution and data selection. 

Training data attribution estimates how each train-ing example affects a prediction, loss, or behavior. Influence functions formalize this idea by measur-ing how upweighting a training example changes a target objective (Koh and Liang, 2017). Later work improves the accuracy, scalability, and stabil-ity of attribution through representer points, gradi-ent tracing, and scalable influence approximations (Yeh et al., 2018; Pruthi et al., 2020; Kwon et al., 2024). These attribution tools provide the scor-ing machinery for many data-selection pipelines, where candidate examples are selected according to their estimated effect on a target objective. In LLM fine-tuning, data selection improves data effi-ciency by identifying examples that are most use-ful for a target behavior. LESS selects influen-tial data for targeted instruction tuning by com-paring candidate examples with a small target set (Xia et al., 2024). IF-GUIDE applies a similar influence-guided view to detoxification by trac-ing undesirable generations back to training data (Coalson et al., 2025). PRISM follows this target-conditioned data-selection paradigm and uses at-tribution to score SFT examples, but it does not introduce a new influence estimator. Instead, it im-proves the target signal that the influence score is aligned with. 

Target behavior representation. Before train-ing examples can be scored, the target behavior must be represented as a direction or signal. Exist-ing methods build such representations in different spaces. Concept Influence and Persona Vectors construct activation-space directions, often by av-eraging representations of examples that express a concept or trait (Kowal et al., 2026; Chen et al., 2025). Influence-based data selection constructs parameter-space directions, often by averaging gra-dients from target examples (Xia et al., 2024). IF-GUIDE further uses a contrastive positive-negative construction in gradient space (Coalson et al., 2025). Although these approaches differ in rep-resentation space, they commonly aggregate target examples uniformly, implicitly treating all exam-ples as equally informative for the current model. PRISM relaxes this assumption by weighting target examples according to the current model’s prefer-ence, producing a preference-aware target direction for training-data scoring. 3 Methodology

Influence-function-based Data Selection Method), a preference-aware influence scoring method for budget-efficient data selection. Section 3.1 gives the motivation and task objective. Section 3.2 then defines the notation and scoring framework. 

3.1 Motivation and Definition 

The goal of PRISM is to identify, under a limited data budget, the candidate SFT examples that have the largest estimated effect on a target behavior. Achieving this requires a precise representation of the target behavior itself. When the target behavior is described by multiple target examples, these ex-amples can differ substantially in how useful they are for guiding the current model: examples closer to the model’s existing behavior often provide more actionable local guidance, while distant examples may provide weaker selection signals. PRISM therefore represents the target behavior in a preference-aware way. It first uses the current model’s preference to distinguish the local useful-ness of different target examples, and then uses in-fluence scoring to estimate how each candidate SFT example affects this preference-aware target behav-ior. The resulting ranking supports both directions of data control: for desired behaviors, high-ranked examples can be retained for efficient fine-tuning; for undesired behaviors, high-ranked examples can be removed for repair.

The same analysis shows why a uniformly weighted contrast can deviate from the preference-aware target di-rection. We state the argument at θ, with positive definite local training Hessian Hθ. 5 Experiments 

5.1 Overview 

We evaluate PRISM from two complementary data-selection directions: forward selection and reverse repair. In the forward-selection setting, we select a small fraction of training data and measure down-stream benchmark performance after fine-tuning. In the reverse-repair setting, we identify and re-move training examples that promote undesirable behavior, then test whether retraining on the re-tained data repairs the model. The experiments address three questions. First, under a fixed 5% data budget, does PRISM se-lect data that improves general benchmark perfor-mance? Second, in the reverse-repair setting, can PRISM effectively identify harmful SFT examples so that filtering and retraining under the same bud-get recovers more aligned benchmark behavior? Third, are the paired-response construction and preference weighting necessary, and is the method stable across deletion budgets? Baselines. Across these settings, we compare PRISM with a shared set of data-selection base-lines whenever applicable. LESS selects influen-tial data for targeted instruction tuning by align-ing candidate examples with target-example gra-dients (Xia et al., 2024). IF-G UIDE follows the influence-function-guided toxic-token attribution method of Coalson et al. (2025); relative to PRISM, it uses the influence-function signal without pref-erence weighting. VEC FILTER , P ROJ DIFF , and CONCEPT INF follow the Vector Filter, Projection Difference, and Concept Influence scores of Kowal et al. (2026), corresponding to activation-only, activation-difference, and gradient-based concept-attribution variants. RANDOM selects a budget-matched subset uniformly at random. Reverse-repair-specific baselines are introduced in Sec-tion 5.3. 

5.2 Forward Selection for Efficient Fine-Tuning 5.2.1 Setup 

We first evaluate utility-oriented forward selection under a fixed 5% data budget. Following the effi-cient fine-tuning setting of Xia et al. (2024), the candidate pool is an instruction-tuning mixture of about 270K examples assembled from FLAN V2, chain-of-thought data, Dolly, and OpenAssistant (Longpre et al., 2023; Wei et al., 2022; Conover et al., 2023; Köpf et al., 2024). These data cover heterogeneous instruction formats and reasoning tasks and contain no obvious in-domain data for the target evaluation queries. Each method selects 

5% of this pool for fine-tuning Llama-2-7B and Llama-2-13B (Touvron et al., 2023). We evaluate the resulting models on MMLU, BBH, TyDi QA, and MATH-500, covering factual knowledge, chal-lenging reasoning, multilingual question answer-ing, and mathematical problem solving (Hendrycks et al., 2021a; Suzgun et al., 2023; Clark et al., 2020; Hendrycks et al., 2021b; Lightman et al., 2024). 

5.2.2 Results and Analysis PRISM-selected data enables more efficient LLM fine-tuning and stronger downstream per-formance. Table 1 shows that PRISM is best on eight of ten model–benchmark rows, including MMLU, TyDi QA, and MATH-500 for both Llama-2 model sizes. This suggests that a preference-aware target direction helps the selected subset pre-serve broadly useful training signals under a small data budget. The main exception is BBH, where LESS or random selection gives the strongest re-sult. One possible reason is that BBH is already divided into fine-grained task categories, and each category contains only a small number of target examples; in this case, there is limited room for preference-aware reweighting to distinguish exam-ples within the same target behavior.

5.4 Ablation Studies on Reverse Repair We ablate PRISM in the same reverse-repair protocol used above. 6 Conclusion 

In this paper, we propose PRISM to improve the budget efficiency of data selection for SFT. PRISM represents a target behavior with model preference, giving larger weights to target examples that are more actionable for the current model, and then traces this preference-aware direction back to can-didate training data through influence scoring. This enables the same ranking rule to support both effi-cient forward selection and safety-oriented reverse repair. Through experiments across model families and scales, PRISM consistently selects data that leads to stronger fine-tuning performance and more effective behavioral repair under fixed budgets. We also provide a first-order theoretical explanation showing why preference-aware target representa-tion improves the selection direction. In this way, PRISM offers a simple perspective on data selec-tion: better target-behavior characterization can make limited SFT data more useful.

A Benchmark-derived Repair Signal 

All data-selection methods are evaluated under the same benchmark-derived repair setting from Sec-tion 5.3. For each mixed dataset, we first fine-tune the model on the unfiltered SFT data and eval-uate both the pre-SFT and post-SFT models on 

mask_provide_fact . We then collect the bench-mark queries whose behavior changes undesirably and pair the pre-SFT response with the post-SFT response on the same query. This shared paired-response set is the only benchmark-derived sig-nal provided to the data-selection methods. Each method converts this signal into a score over the original SFT examples, removes the same fixed fraction of high-risk examples within a setting, and retrains from the same starting model on the re-tained data. Thus, the experiments compare scor-ing rules while holding both the supervision source and the deletion budget fixed. Section 5.3.4 varies the retained data ratio explicitly. 

B Benchmark Degradation from Harmful SFT Data 

Before evaluating data repair, we verify that the data families create the failure mode of interest. Table 4 compares the base model, pure harmful SFT, and pure correct SFT conditions. The pure harmful conditions increase benchmark dishonesty in several model-family pairs, while pure correct SFT generally lowers it. This confirms that the benchmark is sensitive to the harmful-data fami-lies and motivates filtering mixed SFT data before retraining. The main ranking and filtering tables focus on emergent-misalignment settings, i.e., model-family pairs where mixed harmful data produces a clear benchmark degradation relative to the base model. Settings without this degradation are not the pri-mary data-repair target, because there is no ob-served benchmark failure to repair.
