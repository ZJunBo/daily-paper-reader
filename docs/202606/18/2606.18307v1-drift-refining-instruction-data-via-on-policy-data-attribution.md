## 摘要
优化监督微调（SFT）的训练数据分布决定了大型语言模型（LLM）的能力上限。现有数据整理方法在预算有限的情况下能够加速训练，但不太适合提升能力上限。这里的挑战不再是识别一个能保持性能的较小子集，而是将数据分布优化为最能提升最终模型的实例。为解决此问题，我们探索使用影响函数（IF）进行实例级数据归因。我们识别出标准IF公式在此场景中因两种结构限制而表现不佳：由离策略验证目标引起的邻近差距，以及对梯度范数的严重偏差。我们提出DRIFT（通过基于策略的影响函数进行监督微调数据优化）。DRIFT不依赖外部参考数据，而是利用模型自身的基于策略的输出来生成验证目标，这从经验上最小化了参数邻近差距，并更好地符合IF的局部邻域假设。它进一步基于轨迹正确性应用符号加权，并针对梯度欺骗问题去偏影响分数，使得少量验证查询能够作为可靠锚点来归因整个数据集。在7B参数指令和推理模型上的实验表明，DRIFT持续提高了两者的性能上限，优于现有的数据整理基线。

## Abstract
Optimizing the training data distribution for Supervised Fine-Tuning (SFT) dictates the capability of Large Language Models (LLMs). While existing data curation methods excel at accelerating training under constrained budgets, they are less suited to elevating the capability upper bound. The challenge here is no longer to identify a smaller subset that preserves performance, but to refine the data distribution toward instances most capable of improving the final model. To address this problem, we explore instance-level data attribution using Influence Functions (IF). We identify that standard IF formulations struggle in this setting due to two structural limitations: a proximity gap caused by off-policy validation targets, and a severe bias towards gradient norm. We propose DRIFT (Data Refinement via On-Policy Influence Functions for Supervised Fine-Tuning). Instead of relying on external reference data, DRIFT utilizes the model's on-policy rollouts as validation targets, which empirically minimizes the parameter proximity gap and better aligns with the local neighborhood assumption of IF. It further applies signed weighting based on trajectory correctness and debiases influence scores against the gradient hacking issue, allowing a small set of validation queries to act as reliable anchors for attributing the full dataset. Experiments on 7B-parameter instruction and reasoning models show that DRIFT consistently raises the performance ceiling on both, outperforming existing data curation baselines.

- **问题背景**：监督微调（SFT）是大型语言模型（LLM）对齐指令、提升下游能力的关键阶段，训练数据的分布直接影响模型的能力上限。现有数据整理方法主要分为三类：域级数据混合（粗粒度，忽略实例质量差异）、数据选择（面向预算受限场景，追求效率而非性能上限提升）、在线批处理（频繁重估权重，破坏高效训练基础设施）。
- **核心挑战**：当模型已经在完整SFT数据集上充分训练（收敛）后，传统效率驱动的数据选择方法难以继续提升性能天花板。需要一种**实例级数据归因**方法，识别出那些对**已收敛模型**仍有价值、能进一步推动性能提升的样本。
- **研究动机**：影响力函数（Influence Functions, IF）理论上可量化每个训练样本对验证目标的影响，但标准IF在LLM中存在两大结构脆弱性：
  1. **邻近差距**：外部离策略（off-policy）验证目标要求模型发生全局参数偏移，违背IF的局部泰勒展开假设。

# 2. 方法论：核心思想、关键技术细节

## 2.1 核心思想
DRIFT（Data Refinement via On-Policy Influence Functions for Supervised Fine-Tuning）将SFT数据归因转化为**基于策略梯度的对比式影响函数**。关键洞察：用模型自身的生成结果作为IF目标，可保持参数在局部邻域内移动，符合IF的理论假设；通过正/负轨迹的对比加权，近似策略梯度估计器，从而强化有益推理路径、惩罚不良路径。

## 2.2 技术细节（四步流程）

**Step 1：在策略验证目标构造**
- 基于已完全收敛的基座模型 θ*，对少量验证查询 Xval 采样生成多条响应（rollout），形成在策略集 Zrollout。
- 避免使用外部参考响应，从而缩小参数偏移，满足IF的局部假设（如图1、表5验证）。

**Step 2：策略梯度IF形式**
- 将验证目标重定义为**期望奖励最大化**，每个 rollout 赋予二元权重 y_val ∈ {+1, -1}（正确/错误）。
- 梯度近似为 REINFORCE 形式：
  ∇_θ J_PG ≈ (1/N_x) Σ_i y_val^(i) ∇_θ log π_θ*(y^(i)|x)。
- 使用 GraSS 稀疏随机投影将梯度降维至 d 维，得到 ϕ(z; θ*)。对每个任务 k，加权聚合得到验证质心，再与训练样本投影做点积：
  S_raw^(k)(z_train) = ϕ(z_train)^⊤ (Σ_{z_val∈D_k} w(z_val) ϕ(z_val))。

**Step 3：影响分数去偏**
- 原始影响分数与梯度范数呈幂律关系 |S| ∝ ‖ϕ‖^β，且 β 随任务变化。
- 对每个任务独立进行**对数空间正交化**：对 log|S_raw| 与 log‖ϕ‖ 做线性回归，得到斜率 β^(k)，然后计算去偏分数：
  S̃^(k)(z_train) = sign(S_raw^(k)) · exp( log|S_raw^(k)| - β^(k) log‖ϕ‖ )。
- 为防止数值不稳定，掩码梯度范数最小的10%样本。

**Step 4：持续训练与数据重分配**
- 按去偏后总分 S(z_train) 降序排列训练样本，默认选取 top 10% 构成精炼数据集。
- 使用保守的超参数（小学习率 1e-5、长序列打包）对基座模型进行**持续微调**，突破性能饱和。

# 6. 主要结论与发现

1. **DRIFT持续提升性能天花板**：在两个7B模型上，DRIFT在目标域和非目标域均取得平均最佳结果（如Olmo3总平均+0.95 vs 次优+0.42；OpenR1总平均+1.43 vs 次优+0.62）。相比之下，大多数效率导向方法（DSIR、LESS、RDS）无法超越随机基线，甚至造成性能下降。
2. **离策略IF在LLM上脆弱**：标准IF（使用外部参考）因邻近差距和梯度偏差，效果微弱甚至负向，证实了所提问题的严重性。
3. **Self-Distillation不稳定**：直接训练rollout导致严重过拟合和熵坍缩（尤其推理模型OpenR1平均下降近8%），而DRIFT仅将其作为归因锚点，既利用其信号又避免过度拟合。
4. **在策略目标+符号权重+梯度去偏**是有效归因的三个关键支柱。消融实验证明：
   - 多rollout、负样本对比信号均有益。
   - 梯度去偏必须采用每任务对数空间正交化，简单余弦或长度校正效果不佳。
5. **DRIFT检索到的是推理过程相似而非表面词汇相似的样本**（图3），说明其在推理步骤空间而非字面匹配空间进行归因。

# 8. 不足与局限

1. **算力开销未量化**：论文未报告GraSS投影、IF计算、rollout生成等各环节所需的GPU时间或成本，不利于读者评估实用性。
2. **高维梯度偏差的理论来源不清**：虽观察到幂律关系并有效去偏，但未解释LLM梯度空间中这种非线性偏差的根本几何或优化原因，可能掩盖更深层问题。
3. **有限样本验证**：所提定理的有限样本方差界未给出，实际rollout数量如何影响归因稳定性缺少理论保证。
4. **去偏仍不完美**：少量异常样本（如极长/短响应、质量极低样本）仍可绕过去偏获得高分（附录表7），梯度范数并非唯一混淆因素。
5. **模型规模与领域限制**：仅在7B参数、推理和指令两个模型上验证；未探索更大规模（70B+）或其他类型（代码、多模态）模型，泛化性待验证。
6. **验证集选择依赖人工先验**：需手动选择合适（未饱和、推理型）的验证任务，实际部署中可能需反复尝试；一旦选错，归因可能无效。
7. **对比基线未包括最新方法**：如主动学习类在线方法、基于模型困惑度的方法等，主要聚焦三类范式中的一两类。
