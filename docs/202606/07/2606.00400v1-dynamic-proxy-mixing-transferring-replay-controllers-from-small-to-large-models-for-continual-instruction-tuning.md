## 摘要
持续指令微调通过一系列新领域更新语言模型，但每次更新都会逐步侵蚀先前学习的能力和一致性行为。重放是标准的缓解措施，但固定重放比例本质上存在局限性，因为最优混合比例随当前领域、训练阶段以及先前行为不断变化的脆弱性而改变。我们提出PROX-YMIX框架，该框架在小代理模型上学习动态重放控制器，并将冻结的控制器迁移到更大的目标模型。该控制器从不观察未来任务，而是根据归一化验证损失及其时间动态构建状态，在当前任务和可访问的重放缓冲区上生成掩码混合。我们的核心经验假设是遗忘镜像：即使绝对损失幅度不同，任务脆弱性排名在不同模型规模间仍基本一致。在跨规模迁移控制器之前，我们通过实验验证了这一假设。在LLaMA-3-8B上跨越五个持续指令微调序列，PROXYMIX在平均准确率上提升3.4个百分点，最终遗忘降低3.5个百分点，安全得分提升5.8个百分点（相较于最强的非预言机基线），且策略学习成本约为预言机目标强化学习的1/50。该框架在接口层面无信息泄露且与架构无关，同时我们也指出了代理假设失效的情况，突出了稳健部署的局限性。

## Abstract
Continual instruction tuning updates a language model through a sequence of new domains, yet each update can progressively erode previously learned capabilities and alignment behavior. Replay is the standard mitigation, but fixed replay ratios are inherently limited because the optimal mixture varies with the current domain, the training stage, and the evolving vulnerability of prior behaviors. We propose PROX-YMIX, a framework that learns a dynamic replay controller on a small proxy model and transfers the frozen controller to a larger target. The controller never observes future tasks and constructs its state from normalized validation losses and their temporal dynamics, producing a masked mixture over the current task and accessible replay buffers. Our core empirical hypothesis is forgetting mirroring: task vulnerability rankings remain largely consistent across model scales even when absolute loss magnitudes differ. We validate this assumption empirically before transferring controllers across scales. On LLaMA-3-8B across five continual instruction tuning sequences, PROXYMIX improves average accuracy by 3.4 points, reduces final forgetting by 3.5 points, and raises safety score by 5.8 points over the strongest non-oracle baseline, at roughly 50x lower policy learning cost than Oracle Target RL. The framework is leakage free and architecture independent at the interface level, and we also identify settings where the proxy assumption breaks down, highlighting limitations for robust deployment.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：在持续指令微调（Continual Instruction Tuning）场景下，语言模型需要按序列依次适应多个新领域，但每次更新都会导致先前学到的能力与对齐行为（alignment behavior）逐渐退化，即“对齐漂移”（alignment drift）。经典缓解策略是经验重放（experience replay），即混合当前任务数据与旧任务数据进行训练。
- **核心问题**：固定重放比例（如总数据的10%或30%）无法适应不同领域、不同训练阶段以及任务脆弱性的动态变化。最优混合比例需要根据模型当前状态、遗忘风险、安全约束等因素实时调整。直接在大规模目标模型上学习动态重放策略计算成本极高（每个强化学习回合需要一次完整的持续训练轨迹）。
- **整体含义**：本文提出一种代理到目标（proxy-to-target）的迁移框架——PROXYMIX，通过在小代理模型上训练一个动态重放控制器，再冻结并迁移到更大目标模型，使重放策略能够自适应调整，同时大幅降低学习成本。

## 2. 方法论：核心思想、关键技术、算法流程

### 核心思想
- **代理模型学习动态重放策略**：在规模较小的代理模型（如Pythia-160M/410M/1.4B）上，通过近端策略优化（PPO）训练一个策略网络，根据各任务的归一化验证损失（normalized validation loss）及其时间趋势（temporal dynamics）生成重放混合权重。
- **遗忘镜像假设（Forgetting Mirroring）**：假设任务脆弱性排名在不同模型规模间保持一致——即代理模型认为最易遗忘的任务，在目标模型上也往往最脆弱。该假设通过实验验证（Spearman相关系数0.81~0.90）。
- **严格的无泄露训练**：控制器在决策时只能看到当前及之前任务的验证信号，未来任务在状态、动作和奖励中均被掩码（masked），确保符合持续学习的因果性要求。
- **控制器迁移**：学习到的策略冻结后直接应用于目标模型，目标模型仅需计算归一化状态并查询策略，无需再次训练策略。

### 关键技术细节
- **状态表示**：由三部分组成：归一化损失、归一化损失的时间差分、当前任务索引one-hot编码、当前任务内进度。对于未来任务，对应项被掩码为零。
- **归一化**：使用运行估计的均值和标准差对损失进行标准化，消除绝对值差异，使策略可在不同规模模型间迁移。
- **动作空间**：策略输出logits，经掩码softmax生成当前任务与所有历史任务上的重放概率分布（掩码确保未来任务权重为零）。
- **奖励函数**：由三项构成——当前任务损失下降、历史任务损失上升惩罚（取正部分平均）、安全奖励（仅当安全验证信号可用时）。
- **策略优化**：采用PPO算法，每次episode对应代理模型的一次完整持续微调轨迹。策略网络为两层MLP（256隐单元、tanh激活），价值网络共享相同结构。
- **算法流程**（伪代码）：
  1. 随机初始化策略πϕ和批评家。
  2. 对每个episode（共E次）：
     - 将代理模型重置为初始检查点。
     - 按任务顺序T1...TT训练代理模型，每隔δ步执行一次策略决策：
       - 计算当前及所有已见任务的验证损失，构建掩码状态。
       - 策略输出重放混合权重。
       - 使用该混合采样当前批次数据（当前任务+回放数据）更新代理LoRA参数。
       - 计算奖励并存储转移。
     - 完成一个episode后，使用PPO更新策略和批评家参数。
  3. 返回训练好的策略πϕ。

### 数据集与场景
- 指令集涵盖异构领域：代码（Code-Alpaca等）、数学（MetaMath-style）、聊天（ShareGPT）、医疗、法律、创意写作、摘要、问答、安全（有害/良性提示）。共构建5个任务序列（S1~S5），见表1：
  - S1：Code → Math → Chat
  - S2：Chat → Medical → Legal → Code
  - S3：Safety → Code → Writing → Math
  - S4：Summarization → QA → Code → Chat → Math
  - S5：Code → Python → Rust → Math

### 基准方法（Baselines）
- **No Replay**：不进行重放，下限。
- **Random Replay 10%/30%**：均匀随机重放固定比例。
- **Proportional Replay**：按当前验证损失大小分配重放概率。
- **Loss-Slope Replay**：按损失时间差分（上升趋势）分配重放权重。
- **GradSim Replay**：基于梯度余弦相似性选择重放任务。
- **InsCL**：基于指令相似性动态重放（SOTA动态重放基线）。
- **O-LoRA**：正交低秩子空间保护方法。
- **DoReMi Static**：在代理模型上学习固定混合权重后迁移（静态混合）。
- **Oracle Target RL**：直接在目标模型上训练PPO策略，作为上界参考。

### 评估指标
- **AA**（平均准确率）：所有任务最终准确率均值。
- **FGT**（最终遗忘量）：每个任务相比其最佳历史点的平均下降。
- **NTA**（新任务准确率）：各任务刚学完时的准确率均值。
- **BWT**（反向迁移）：最终准确率减去刚学完时的准确率。
- **SS**（安全得分）：有害提示拒绝准确率减去γ倍良性过度拒绝惩罚的平衡分数。

## 6. 主要结论与发现

1. **PROXYMIX显著优于所有非上界基线**：在LLaMA-3-8B上，相比最强非Oracle基线（InsCL）：平均AA提升3.4点（66.87 vs 63.5），FGT降低3.5点（8.63 vs 12.1），SS提升5.8点（85.83 vs 80.0）。
2. **接近Oracle目标RL性能**：PROXYMIX的AA（66.87）接近Oracle（68.1），FGT（8.63 vs 7.1），SS（85.83 vs 87.0），但训练成本仅为Oracle的约1/50。
3. **遗忘镜像假设成立**：Spearman相关性在0.81~0.90之间，说明任务脆弱性排名跨尺度一致。
4. **跨架构迁移有效**：单一Pythia-410M控制器迁移到LLaMA-3-8B和Mistral-7B均获得显著增益（AA提升+9.3和+8.3）。
5. **安全关键序列鲁棒**：在S3上，PROXYMIX实现81.2%有害拒绝率，同时保持88.3%良性合规，远超其他基线。
6. **控制器行为可解释**：策略可视化显示，控制器在安全阶段后主动增加安全重放，并在写作阶段（可能干扰拒绝行为）再次提升安全性重放，表现出预期性而非纯反应式。

## 7. 优点

- **高效的学习成本**：通过代理模型摊销策略学习成本，实现约50倍效率提升，使动态重放控制实用化。
- **严格因果掩码设计**：未来任务在状态、动作、奖励中均被掩码，确保训练和评估符合持续学习真实约束，避免泄露作弊。
- **遗忘镜像假设的经验验证**：将跨尺度迁移建立在对假设的定量验证基础上，而非隐含假设，增强方法可信度。
- **归一化状态表示**：消除损失绝对值差异，使策略无需调整即可在不同规模、不同架构模型间零样本迁移。
- **安全性显式优化**：在奖励中包含安全项，分离有害拒绝与良性合规的指标（安全得分SC定义），避免仅优化有害拒绝导致过度拒绝。
- **鲁棒性分析全面**：涵盖代理规模、验证噪声、LoRA模块选择、验证间隔、重放平衡方式等多种扰动，显示方法在一定范围内稳健。
- **跨模型家族验证**：除同族Pythia外，还迁移到LLaMA-3和Mistral，证明方法不依赖特定架构。

## 8. 不足与局限

- **代理假设失效场景**：在专业领域（如医疗、法律）中，代理模型缺乏足够表征覆盖，导致损失轨迹无信息；此外，大模型可能出现小模型没有的跨任务交互（如数学推理与代码生成），此时遗忘镜像减弱。论文明确分析了这些失败模式。
- **依赖验证信号质量**：控制器只能保护验证流所覆盖的行为；若安全评估集未覆盖某些攻击模式，则无法保护。这要求精心设计验证流。
- **有限任务序列长度**：策略预先设定最大任务数T，并通过掩码适应较短序列，但无法直接扩展至更长或可变长度序列，需要重新训练或引入分层控制。
- **重放依赖于历史数据存储**：需要保存或再生历史任务数据，可能涉及隐私、数据治理和许可限制，论文承认未解决这些实际约束。
- **实验种子数仅3**：虽然使用了匹配种子以减少随机混杂，但3个种子对优化方差估计有限，某些指标（如安全得分）可能对初始化敏感。
